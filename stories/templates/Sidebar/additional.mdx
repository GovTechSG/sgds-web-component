## Level 1 only

SidebarItem is used as a standalone top-level navigation item. Level 1 items are shown directly in the sidebar and do not have nested children. They can be terminal leaf nodes in your navigation structure.

**Requirement:** Level 1 items must have a leading icon in the `leadingIcon` slot. Trailing icon is optional.

## With nested items (Level 2)

SidebarGroup is used for parent navigation items that contain nested children. Level 2 groups display their items in a drawer overlay when the parent group is clicked, allowing for expandable menu hierarchies.

**Requirement:** Level 1 and 2 items/groups must have leading icons in the `leadingIcon` slot. Trailing icon is optional for all levels.

<Canvas of={SidebarStories.Level1Level2}>
  <Story of={SidebarStories.Level1Level2} />
</Canvas>

## Double nested structure (Level 3)

SidebarGroup supports up to two levels of nesting, creating three-level navigation hierarchies. Nested groups show their items in a drawer when Level 1 groups open, making complex navigation structures manageable.

**Requirement:** For Level 3, leading icon is optional. Trailing icon is optional for all levels.

## Collapsed state

The sidebar can be collapsed by setting the `collapsed` property to `true`. When collapsed, only icons are visible for Level 1 options. Nested options are still accessible via the drawer overlay.

## With sections

SidebarSection is used to organize sidebar items into logical groups. Sections are collapsible containers that can hold both items and groups, enabling a well-organized navigation structure with clear separation between different parts of your application.

## Active state

The sidebar tracks the currently active item via the `active` property. When an item or group is selected, its `name` property is set as the active value. This is useful for highlighting the current page or section in your application, providing clear user feedback about their location.

## Link navigation

SidebarItem and SidebarGroup can wrap anchor elements to enable programmatic navigation. When an item with an anchor link is clicked, the link is automatically followed, allowing the sidebar to work seamlessly with your application's routing system.
