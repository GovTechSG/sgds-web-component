## Validation

HTML form constraint validation is built-in. By default, there is no need to control the `invalid` prop to enable constraint validations. 

- To enable invalid styles and feedback message set `hasFeedback` to true

Press submit to show constraint validation behaviour for `required` and `minlength=5`. 
Resetting the form, reset's the validity and the value of the input to its initial default value.

### Overriding the invalid feedback message 

By default, the invalid feedback reflected is based on the browser's native error message based on the ValidityState.  To override the default message, 
use `invalidFeedback` prop.

## Default Value 

SgdsTextarea has an internal property that saves the initial value of `value` as the default value of the control.
On form reset, the control defaults to the initial value of control.

Test this out by following the instructions below 
1. Change the value of textarea below by typing in it and then click the reset button.
    - Observe how the value resets to the initial value on load
2. Click on the yellow button to change the defaultValue of the textarea
3. Click reset button again 
    - Observe now the default value has changed 

<Canvas of={TextareaStories.DefaultValue}>
  <Story of={TextareaStories.DefaultValue} />

```jsx
export const changeValue = (e) => {
  e.preventDefault()
  const textarea = document.querySelector("sgds-textarea#default-value-eg")
  textarea.defaultValue = "Default value has changed!"
}
```
